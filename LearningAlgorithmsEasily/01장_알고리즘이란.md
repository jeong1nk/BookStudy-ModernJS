# 01. 알고리즘은 문제 해결 과정을 묘사하는 것
- **🏷️알고리즘: 어떤 작업을 수행하기 위해 입력을 받아 원하는 출력을 만들어내는 과정을 기술한 것.**
- 알고리즘을 설계하기 위해서는 우선 "무엇을" 하는지를 입력과 출력으로 명시할 수 있다.
<br />

- 예) 학생 100명의 시험점수를 입력으로 받아 최고점으로 출력하는 작업을 한다.
- 입력: 100개의 점수
- 출력: 입력된 100개의 점수 중 최댓값
<br />

- 위의 예제를 좀 더 구체적으로 표현할 수도 있다.
- 입력: 100개의 변수 x[1], ..., x[100]의 값
- 출력: x[1], ..., x[100] 중 최댓값
<br />

- 위의 입출력을 요구하는 알고리즘은 대략 형태가 아래와 같을 것이다.

```
maxScore(x[], n){
  x[1...n]의 값을 차례대로 보면서 최댓값을 계산한다;
  return 위에서 찾은 최댓값; 
}
```

- 이렇게 **문제들에 대한 해결 과정을 모호하지 않게 묘사하는 것이 알고리즘이다.**
- 알고리즘은 모호해서는 안된다. **알고리즘은 명확하고 효율적이어야 한다.**


# 02. 알고리즘은 생각하는 방법을 훈련하는 것
- 알고리즘을 명확하고 효율적으로 설계하고, 알고리즘의 효율적으로 분석하는 작업을 하게 될 것이다.
- 이러한 훈련을 통해 어떤 문제를 해결하기위해 중요한 것과 중요하지 않은 것을 구분할 수 있고, 또한 문제를 단순화해서 볼 수 있다. 


# 03. 알고리즘은 자료구조의 확장
- 자료구조는 생각의 전개를 위해 필요한 기본적인 도구다.
- 예) 건물을 지을때 사용하는 기본적인 건축 재료나 모듈, 자동차 제작의 기본 부품 등
- 자료구조에 대한 기본적인 지식이 있어야 알고리즘을 이해할 수 있을 것이다.
