- 📌점근적 표기법 이해


# 01. 몇 가지 기초 사항들
### 1. 알고리즘 분석의 필요성
- 알고리즘을 설계하고 나면 만든 알고리즘이 자원을 얼마나 소모하는지 소요 시간이 얼마나 걸리는지 분석해야 한다.
- 시간 분석은 최악의 경우와 평균적인 경우에 대한 분석이 대표적이다.
- 시간 분석을 통해 알고리즘이 어느 정도 입력에서 어느 정도의 시간이 필요한지 미리 짐작해, 주어진 시간에 요구하는 작업을 완료할 수 있을지 대략 짐작할 수 있다.

### 2. 알고리즘의 수행 시간 
- 알고리즘의 수행 시간은 입력의 크기에 대해 시간이 어떤 비율로 소요되는지로 표현한다. 
- 보통 문제의 크기가 클수록 걸리는 시간 소요와 차이가 크다.





### 3. 재귀(자기호출)와 귀납적 사고
- 🏷️귀납적 사고: 성격이 같지만 크기가 다른 문제 간의 "관계"를 파악하는 것
- 🏷️수학적 귀납법: 자신보다 작은 문제에 대해 결론이 옮음을 가정하고 자신과 이 작은 문제의 관계를 통해서 자신에게도 결론이 옮음을 보이는 것

- 자기호출을 이용하는 알고리즘의 정당성은 수학적 귀납법과 관련이 깊다.
- 재귀 알고리즘이 자신보다 작은 문제를  자기호출 하는 것은 수학적 귀납법에서 자신보다 작은 문제에 대해 결론이 옮음을 가정하는 것이다. 즉, 자신보다 작은 문제에서는 이 알고리즘이 제대로 작동한다고 가정하는 것이다.
- 자기호출을 이용하는 알고리즘에서 자기호출을 제외한 나머지 부분은 대부분 크기가 다른 문제 간의 관계를 반영하는 것이다.

- 아래 예제는 자기호출과 관계의 반영에 관련된 대표적인 자기호출 알고리즘인 병합 정렬이다. 




# 02. 점근적 표기
- 알고리즘의 효율성이 문제가 되는 것은 입력의 크기가 충분히 클때다. 따라서 알고리즘의 수행시간은 항상 입력 크기가 충분히 클 때 분석한다. 즉, 🏷️점근적 분석을 한다.
- 이때 변수의 크기가 충분히 큰 경우에 변수가 커짐에 따라 함수가 증가하는 비율을 표현하는 방법의 함수를 🏷️점근적 증가율 이라고,  그표기법을 🏷️점근적 표기법이라고 한다.
