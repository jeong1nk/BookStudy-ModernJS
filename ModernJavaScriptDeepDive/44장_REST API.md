- REST는 HTTP/1.0과 1.1의 스펙 작성에 참여했고 아파치 HTTP 서버 프로젝트의 공동 설립자인 로이 필딩의 2000년 논문에서 처음 소개되었다.
- HTTP의 장점을 쵣한 활용할 수 있는 아키텍처로서, HTTP 프로토콜을 의도에 맞게 디자인하도록 유도하고 있다.
- **REST의 기본원칙을 성실히 지킨 서비스 디자인을 "RESTful"이라고 표현한다.**
<br />

- 🏷️**REST(REpresentational State Transfer): HTTP를 기반으로 클라이언트가 서버의 리소스에 접근하는 방식을 규정한 아키텍처**
- 🏷️**REST API: REST를 기반으로 서미스 API를 구현한 것**

# 44.1. REST API의 구성
- REST API는 자원, 행위 표현의 3가지 요소로 구성된다,
- REST는 자체 표현 구조로 구성되어 RET API만으로 HTTP의 요청의 내용을 이해할 수 있다.
| 구성 요소 | 내용 | 표현 방법 |
| ------------------ | ------ | ------ |
| 자원(resource) | 자원 | URI(엔드포인트) |
| 행위(verb) | 자원에 대한 행위 | HTTP 요청 메서드 |
| 표현(representations) | 자원에 대한 행위의 구체적인 내용 | 페이로드 |
